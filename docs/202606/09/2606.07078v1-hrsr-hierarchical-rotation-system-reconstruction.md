## 摘要
当同时需要几何保真度和拓扑控制时，从点云进行曲面重建仍然具有挑战性。旋转系统重建（RsR）通过欧拉示性数显式控制拓扑，从点云重建三角形网格，但其顺序边插入限制了可扩展性。我们提出层次化旋转系统重建（HRsR），通过边折叠和顶点分裂的层次化流程加速RsR。HRsR首先使用k近邻图简化输入，在简化结构上进行重建，然后在保持拓扑的同时恢复几何细节。为了保持几何一致性，我们引入了相交处理和基于质量的顶点分裂选择。实验表明，相比RsR，HRsR实现了高达6倍的加速和超过8倍的内存使用减少，同时实现了可比较的重建结果。

## Abstract
Surface reconstruction from point clouds remains challenging when both geometric fidelity and topology control are required. Rotation System Reconstruction (RsR) reconstructs triangle meshes from point clouds while explicitly controlling topology through the Euler characteristic, but its sequential edge insertion limits scalability. We present Hierarchical Rotation System Reconstruction (HRsR), which accelerates RsR through a hierarchical pipeline of edge collapses and vertex splits. HRsR first simplifies the input using a $k$-nearest neighbor graph, performs reconstruction on the reduced structure, and then restores geometric detail while preserving topology. To maintain geometric consistency, we incorporate intersection handling and quality-driven vertex split selection. Experiments demonstrate up to a $6\times$ speedup and more than $8\times$ reduction in memory usage over RsR, while achieving comparable reconstruction results.

### 1. 核心问题与整体含义（研究动机和背景）
- **问题**：从点云进行三角网格曲面重建时，需要同时保证几何保真度（保留原始点位置）和拓扑正确性。现有组合方法（如 Rotation System Reconstruction, RsR）通过欧拉示性数显式控制拓扑，但顺序边插入导致在大规模点云上计算和内存开销巨大，可扩展性差。
- **动机**：改善 RsR 的效率瓶颈，使其能处理更大规模点云，同时保持拓扑可控和几何一致性。
- **整体意义**：提出层次化加速框架，使拓扑可控的组合重建方法变得实用，并可作为其他组合重建方法的通用加速策略。

### 2. 方法论：核心思想、关键技术细节、算法流程（文字说明）
- **核心思想**：采用层次化边折叠与顶点分裂。先在简化图上进行 RsR 重建，再逐步恢复细节，从而避免在原始高密度点云上执行昂贵的顺序插入。
- **关键步骤**：
  1. **边折叠（Edge Collapse）**：
     - 构建 kNN 图（k 较小，如 6），按边长升序折叠边，直到保留原顶点数的一定比例（collapse ratio p，如 50%/25%/12.5%）。
     - 约束：禁止折叠法向夹角 >90° 的边，防止不同表面合并；新顶点位置/法向为加权平均。
     - 存储折叠历史（对偶信息供后续顶点分裂使用）。
  2. **简化后重建**：对简化点云运行标准 RsR，获得粗网格。
  3. **顶点分裂（Vertex Split）**：
     - 按折叠逆序恢复顶点。每一步从顶点 u 分裂出 unew，引入新边 {uold, unew}，并重新分配邻接。
     - **质量驱动的选择**：对候选 1-ring 边界对，用能量函数 E = ED + wEA 评估：
       - ED = Σ⟨ni,nj⟩−1 ² * 共享边长（惩罚法向不连续）；
       - EA = Σ ℓmin(t) * (π/3 − θmin(t))（惩罚小角）。
       - 选择最优对，并约束最大二面角。
     - **相交处理**：检测新边与 1-ring 边界的交叉，若交叉则翻转（flip）边界边；若涉及边界边无法翻转则移除该边界边。
     - 可在分裂前设置最小边长阈值 lmin，控制输出分辨率。
  4. **局部优化**：每次分裂后，对邻域边进行翻转，基于角度准则（所有角 <90° 且存在角小于阈值）改善三角质量。

### 6. 主要结论与发现
- **效率提升显著**：HRsR 在 12.5% collapse ratio 下实现最高 6.47× 加速（Church 模型），内存降低 7.5× 以上（如 Church 从 155 GB 降至 18.3 GB）。
- **重建质量相当**：在多数场景下，HRsR 的重建结果与 RsR 视觉上接近，且保持了高顶点引用率（>99.9%），优于 Co3Ne（无平滑版失败，有平滑版引用率略低且有更多边界边）。
- **附加特性**：边缘折叠引入了“孔洞填充”效果（可恢复缺失区域），但可能误封闭精细结构（如镂空栏杆）。提供密度过滤能力，通过设置 lmin 可控制输出点密度。
- **限制**：过高 collapse（如 2.7%）会导致高曲率区域重建失败；超过 5 次迭代的边折叠收益递减。

### 7. 优点：方法或实验设计亮点
- **方法层面**：
  - 层次化加速思路简单有效：先简化再重建最后恢复，将 RsR 的 O(N²) 主导开销降低到近似 O(pN) 级别。
  - 顶点分裂采用能量函数（法向连续+小角惩罚）自动选择最佳邻接分配，避免手动调整。
  - 相交检测与边翻转结合，在保持拓扑不变的前提下消除自交。
- **实验设计**：
  - 在超大规模点云（最大 20M）上验证，且报告了实际耗时和内存峰值，具有实用参考价值。
