## 摘要
我们提出SHELLS（通过分层局部采样进行语义头部估计），一种高效的、从前馈多视图图像中进行三维头部重建并建立密集语义对应关系的框架。现有方法通常通过局部特征体素独立优化顶点，这种方法将内存密集的特征采样与网格分辨率耦合，限制了稠密拓扑（>10k顶点）的可扩展性并引入表面噪声。相比之下，SHELLS通过分层采样策略将特征提取与网格分辨率解耦。我们使用带有LoRA适配的DINOv2骨干网络提取多视图特征，投影采样稀疏全局特征云，并预测中间粗网格。该粗先验指导构建分层的、表面感知的采样壳，作为最终重建的离散搜索空间。SHELLS在保持表面一致性的同时，相比基于体素的方法，推理GPU内存减少88%（2.4GB对比20GB）。对于18k顶点网格，中值配准误差降低21%至29%，推理速度提升3.5倍（0.08秒对比0.29秒）。值得注意的是，我们的模型仅使用合成数据训练，却能有效泛化到真实世界捕捉，消除了先前工作中常见的昂贵的预注册多视图数据集需求。

## Abstract
We present SHELLS (Semantic Head Estimation via Layered Local Sampling), an efficient feed-forward framework for 3D head reconstruction in dense semantic correspondence from multi-view images. Existing methods typically refine vertices independently via localized feature volumes. This approach couples memory-intensive feature sampling to mesh resolution, which limits scalability for dense topologies (> 10k vertices) and introduces surface noise. In contrast, SHELLS decouples feature extraction from mesh resolution via a hierarchical sampling strategy. We extract multi-view features using a DINOv2 backbone with LoRA adaptation, projectively sample a sparse global feature cloud, and predict an intermediate coarse mesh. This coarse prior guides the construction of layered, surface-aware sampling shells that serve as a discrete search space for the final reconstruction. SHELLS maintains surface consistency while using 88% less inference GPU memory (2.4GB vs. 20GB) than volumetric baselines. It reduces median registration error by 21% to 29% with a 3.5x inference speedup (0.08s vs. 0.29s) for 18k-vertex meshes. Notably, our model is trained exclusively on synthetic data yet generalizes effectively to real-world captures, eliminating the need for the costly, pre-registered multi-view datasets common in prior work.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：传统多视图三维头部重建依赖MVS扫描后注册，过程耗时、需手动清理且易产生噪声。现有前馈方法（如ToFu、TEMPEH）使用密集体素特征采样，将特征采样与网格分辨率耦合，导致高密度网格（>10k顶点）时内存消耗巨大（~20GB GPU）且引入表面噪声；此外，独立逐顶点优化缺乏全局理解，在遮挡区域易产生伪影。
- **目标**：设计一种高效、拓扑一致、可扩展至稠密网格的多视图头部重建框架，且仅需合成数据训练即可泛化至真实场景。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：粗引导分层表面采样。通过两级预测：先预测粗网格，再以其为指导构建表面感知的分层采样壳，作为最终精细重建的离散搜索空间，从而解耦特征采样与网格分辨率。
- **关键技术细节**：
  - **特征提取**：共用DINOv2骨干（含LoRA适配层，秩r=5），处理每张图像输出特征图；通过4层拼接和1×1卷积+像素重组，最终特征维度98，空间尺寸27×18（输入384×256）。
  - **粗网格预测（图阶段）**：
    - 定义稀疏点云S_g（16个同心二十面体壳，共2592点），无初始位置先验。
    - 将每个点透视投影到各特征图，双线性采样，融合均值与方差得全局特征点云。
    - 下采样模板（3000顶点）经MLP得模板token；全局特征加几何位置编码后经线性层得特征token；两者拼接后由XCiT Transformer处理（自注意力在特征维度，减少计算量）。
    - 粗顶点坐标由注意力加权和（query来自模板token，key来自特征token，value为S_g坐标）回归。
  - **精细网格预测（壳阶段）**：
    - 利用粗网格法线方向，在±4mm处构建分层采样壳（每个粗顶点对应3个采样点，共9000点）。
    - 采用可见性感知特征融合（TEMPEH方法）：根据顶点法线与视线方向点积、深度缓存可见性计算权重，融合加权均值与方差。
    - 静态模板壳（基于下采样模板）提供几何位置编码，与壳特征相加后输入共享权重的XCiT Transformer。
    - 最终顶点由注意力加权和（query来自全分辨率模板token，key来自壳特征token，value为动态壳坐标）回归。
  - **损失函数**：顶点-顶点（V2V）损失 + 顶点-平面（V2P）损失，均采用空间加权（面部特征区域权重5.0/3.0，其余1.0）。无显式拉普拉斯正则化（实验证明无增益）。
- **算法流程**：输入多视图图像→DINOv2+LoRA提取特征→图阶段采样稀疏点云→Transformer预测粗网格→粗网格法线构建壳→壳阶段采样特征→共享Transformer预测精细网格→损失计算（端到端训练）。

## 3. 实验设计
- **数据集**：
  - **合成训练集**：使用Blender Cycles渲染，13个相机视图（分辨率1536×1024），基于内部注册网格（17821顶点，>2500身份）生成，含随机服装、毛发、背景。共300k样本，2064身份，按80%/10%/10%分为训练/验证/测试（身份不重叠）。
  - **真实测试集**：13相机系统的多视图捕捉，9617帧，303个被试，约50种静态表情。参考几何通过深度MVS扫描+3DMM配准获得。
- **基准方法**：
  - TEMPEH（在相同合成数据上重新训练，1.4M步，约30天，单卡H100）。
  - 3DMM回归（单视图参数预测后平均）。
  - 3DMM拟合（优化方法，约35秒）。
- **评价指标**：
  - 顶点-顶点（V2V）距离（对合成数据有GT；对真实数据用参考注册）。
  - 点-表面（P2S）距离（对真实数据用MVS扫描）。
  - 网格质量：三角形面积畸变、三角形翻转率。
- **消融实验**：共8组，覆盖特征提取（替换特征网络、去除下采样、降低特征维度、去除LoRA）和网格预测（改变采样密度、粗网格分辨率、仅粗阶段）。

## 4. 资源与算力
- **训练**：单张NVIDIA H100 80GB HBM3 GPU，批量大小3，训练900k步（约2周）。前500k步只训练粗阶段和LoRA，后400k步端到端。
- **推理**：0.08秒，GPU内存约2.4GB（对比TEMPEH约20GB，减少88%）。
- **TEMPEH基线**：单张H100训练约30天（1.4M步），推理约0.29秒。

## 5. 实验数量与充分性
- **实验数量**：综合实验包括合成数据评测、真实数据评测、消融实验（8组）、网格质量对比、多视图数量鲁棒性测试。
- **充分性**：实验设计较全面，覆盖不同数据源、多种方法和多维度指标。消融实验量化了每个组件的贡献。但真实数据上的参考注册存在偏差（因与3DMM拟合初始化相同），论文中已明确指出此局限性。此外，仅与TEMPEH进行完全一致的训练对比，与3DMM基线对比受限于后者的固有缺陷。总体而言，实验客观且充分。

## 6. 主要结论与发现
- SHELLS在合成数据上中值V2V误差比TEMPEH低29%（1.22mm vs 1.71mm），在真实数据上低21%（1.50mm vs 1.90mm）。
- 推理速度提升3.5倍（0.08s vs 0.29s），GPU内存减少88%。
- 网格质量更优：三角形平均畸变0.38 vs 0.55，三角形翻转率0.08% vs 0.15%。
- 仅用合成数据训练即可泛化到真实多视图捕捉，无需真实预注册数据。
- 对输入视图数量鲁棒（2视图仍可重建）。

## 7. 优点
- **内存高效**：通过稀疏分层采样（共11592个采样点）替代体素密集采样，训练内存约20GB（TEMPEH约65GB）。
- **快速推理**：0.08秒实现18k顶点网格注册，适合实时或大尺度数据集处理。
- **拓扑一致性**：输出固定拓扑网格，支持跨身份、跨表情的密集语义对应。
- **强泛化性**：合成数据训练即可用于真实场景，避免昂贵的预注册真实数据准备。
- **全局表面连续性**：Transformer注意力机制提供全局感知，减少表面噪声和局部伪影。
- **鲁棒性**：对遮挡（头发、口腔）和不同输入视图数表现稳定。

## 8. 不足与局限
- **极端表情失败**：对舌头极端姿态重建不佳，归因于合成训练集中舌头多样性不足。
- **细节缺失**：18k顶点网格仅捕获中频几何，无位移图或纹理网络，缺乏细皱纹和毛孔等高频细节。
- **外部体积**：模型预测皮肤表面，而非外部毛发/衣物体积；若需神经网络体，需扩展合成数据集。
- **单视图欠定**：虽然可处理最少2视图，但单视图重建不适用。
- **训练成本**：虽比TEMPEH低，但仍需两周单卡H100训练；对资源有限研究者可能门槛较高。
- **真实数据集偏差**：真实测试的参考注册与3DMM拟合有依赖，可能对拟合基线有偏。
